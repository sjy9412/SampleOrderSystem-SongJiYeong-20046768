# 반도체 시료 생산주문 관리 시스템 — PRD

---

## 메인 메뉴

기능별 선택 화면을 제공하며, 전체 시료에 대한 요약 정보를 확인할 수 있다.

| 메뉴 | 설명 |
|------|------|
| 시료 관리 | 새로운 시료 등록, 목록 조회, 이름 검색 |
| 주문 (접수 / 승인 / 거절) | 고객 주문 접수 및 생산 라인 담당자의 승인·거절 처리 |
| 모니터링 | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 출고 처리 | CONFIRMED 상태 주문에 대해 출고 실행 |
| 생산 라인 | 현재 생산 중인 시료 및 대기 중인 생산 큐 확인 |

---

## 시료 관리

시료(Sample)는 이 시스템의 가장 기본이 되는 단위다.  
각 시료는 고유한 이름과 속성을 가지며, **시스템에 등록된 시료만 주문 가능**하다.

> **수율** = 정상적인 시료 수 ÷ 총 생산 시료 수

| 메뉴 | 설명 |
|------|------|
| 시료 등록 | 새로운 시료를 시스템에 추가 (속성: 시료 ID, 이름, 평균 생산시간(단위: min), 수율). 시료 ID는 등록 순서대로 S-001, S-002, … 형태로 자동 부여 |
| 시료 조회 | 등록된 모든 시료 목록 확인 (현재 재고 수량 포함) |
| 시료 검색 | 이름 등 속성으로 특정 시료를 검색 |

### 시료 중복 방지

**이름, 평균 생산 시간, 수율이 모두 동일한 시료는 중복으로 간주**하여 등록을 거부한다.  
세 속성 중 하나라도 다르면 별개의 시료로 취급한다.

- 중복 등록 시도 시 "이미 등록된 시료입니다." 오류 메시지를 표시한다.
- 시료 ID는 시스템이 자동 부여하므로 중복 판별 기준에 포함하지 않는다.

---

## 시료 주문

고객이 시료를 요청하면 주문 담당자가 주문을 생성한다.

### 시료 예약

고객이 원하는 시료와 수량을 주문한다. 주문 생성 시점의 상태는 **RESERVED**.

**입력 항목**

| 항목 | 설명 |
|------|------|
| 시료 ID | 주문할 시료의 고유 식별자 |
| 고객명 | 주문을 요청한 고객 이름 |
| 주문 수량 | 요청 수량 (단위: ea) |

---

## 주문 승인 / 거절

메뉴 진입 시 **RESERVED 상태의 주문 목록**을 먼저 표시한다.  
주문 ID를 선택한 뒤 **승인** 또는 **거절**을 선택한다.

### 승인 흐름

| 단계 | 조건 | 처리 결과 |
|------|------|-----------|
| 1차 승인 | 재고 충분 | 즉시 **CONFIRMED** 상태로 전환 |
| 1차 승인 | 재고 부족 | 재고 부족 안내 후 **재확인(승인/거절)** 요청 |
| 재확인 승인 | — | 생산 라인에 자동 등록 후 **PRODUCING** 상태로 전환 |
| 재확인 거절 | — | 즉시 **REJECTED** 상태로 전환 |

### 거절 흐름

1차 선택에서 거절하면 즉시 **REJECTED** 상태로 전환된다.

## 모니터링

담당자가 현재 시스템 상태를 한눈에 파악할 수 있도록 구성된 화면이다.

### 주문량 확인

현재 상태별 주문 목록을 확인한다. REJECTED는 유효한 주문이 아니므로 제외한다.

- 확인 대상 상태: `RESERVED` / `CONFIRMED` / `PRODUCING` / `RELEASE`

### 재고량 확인

각 시료별 현재 재고 수량 및 상태를 확인한다.

| 상태 | 조건 |
|------|------|
| 여유 | 주문 대비 재고 충분 |
| 부족 | 주문 대비 재고 수량 부족 |
| 고갈 | 재고 수량이 0 |

---

## 생산 라인

생산 라인에 대한 정보를 표시한다.  
주문 부족분을 생산하되, **수율 및 오차를 고려**하여 실제 생산량을 산정한다.

**생산량 산정 공식**

```
실 생산량  = ceil(부족분 / 수율)          # 단위: ea
총 생산시간 = 평균 생산시간 × 실 생산량   # 단위: min
```

생산 완료 시 주문 상태: `PRODUCING` → `CONFIRMED`

### 생산 현황

현재 생산 중인 시료에 대한 정보를 표시한다.  
표시 항목은 자율적으로 결정 (예: 주문 정보, 현재까지의 생산량 등).

### 대기 주문 확인

생산 큐에서 대기 중인 작업 목록을 출력한다.

- 스케줄링 전략: **FIFO** (선입선출)

---

## 출고 처리

재고가 충분해진 **CONFIRMED** 상태의 주문에 대해 출고를 처리한다.  
특정 주문을 선택해 출고를 실행하면 주문 상태가 **RELEASE**로 전환된다.
